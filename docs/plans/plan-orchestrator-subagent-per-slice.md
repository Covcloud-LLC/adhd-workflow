> OpenAI: gpt-5.5 · Effort: high
> Claude: claude-opus-4-8 · Effort: high

> Why: automate the purely-mechanical between-slice work (commit, advance) while making slice verification *stronger* than the human-paced flow it replaces — a machine witness, not a self-report.
> Reasoning: `docs/notes/plan-orchestrator-subagent-per-slice-reasoning.md` — its decision (serial, two subagents per slice, gate as a **script** the orchestrator runs) is the plan constraint. Do not re-litigate parallelism, the DAG, or worktrees; all three are explicitly rejected there.

## Definition of done

A new `/run-plan <plan>` skill drives a plan's open slices to completion with no human in the loop, and cannot mark a slice ` ✅` on any agent's say-so. Per slice it spawns two subagents — one authors the check, one implements — and the **orchestrator itself** runs `scripts/slice-gate.sh`, obeying its exit code without interpretation. Any red, missing check, dirty tree, or blocked subagent halts the run with the tree intact and a resume point in `git log`. It has been smoke-run in this repo and driven one real two-slice plan in `workbench`, including a deliberate mid-run failure and resume.

## Acceptance criteria

- [ ] `/run-plan <plan>` refuses to start in a repo with no whole-tree check, on a dirty tree, or on a plan whose slices lack executable `Verify:` commands — naming which slice failed validation, before running slice 1.
- [ ] Per slice: agent A authors the check only; the orchestrator observes **red**; agent B implements; the orchestrator observes **green**, whole-tree **green**, and that B did not modify A's check files. Only then is ` ✅` written.
- [ ] No subagent writes the plan file, invokes `/wrap-up`, runs the gate, or reports a verdict the orchestrator acts on.
- [ ] A red gate halts the run immediately, leaves the working tree untouched, never retries, and reports the slice id plus the failing command.
- [ ] After a halt, `/run-plan <plan>` resumes from `git log` on a clean tree and refuses to resume on a dirty one.
- [ ] Each slice lands as one local commit tagged with its slice id. Nothing is pushed.
- [ ] Each ` ✅` carries its provenance — the command and commit sha that witnessed it.
- [ ] `/standup` and `/audit-plans` ignore plan files whose basename starts with `_`.

### RUN-5 - Write the `/run-plan` skill

> task: Doc/skill-prose slice; red-gate exemption applies because the skill contract is the output, and its behavior is exercised by RUN-6 and RUN-7. Create `skills/run-plan/SKILL.md` per `docs/notes/plan-orchestrator-subagent-per-slice-reasoning.md`. Frontmatter `description` must name the triggers `/run-plan`, "run this plan", "execute the plan", and cross-link `[[pjm]]`, `[[wrap-up]]`, `[[standup]]`. The skill: (1) **Validates before running slice 1** — the plan must carry a whole-tree `Check:` header, every open slice must have a `Verify:` that is a runnable command and a Check/Build split; the working tree must be clean; the repo must have a whole-tree check that can exit non-zero. On any failure, name the offending slice and refuse. (2) **Per slice, serially**: spawn subagent A with the slice's **Check:** text only — A authors the check, implements nothing, and returns; the orchestrator runs `scripts/slice-gate.sh preflight` and requires red; the orchestrator commits A's check alone. Then spawn subagent B with the **Build:** text; B implements; the orchestrator runs `scripts/slice-gate.sh postflight`, which asserts check-green, tree-green, and that B did not touch A's files. Then the orchestrator commits and appends ` ✅ (<check-command>, <sha>)` to the slice heading. (3) **Invariants, stated explicitly in the skill**: the orchestrator never reads a diff or a subagent's full output — only slice id, exit code, sha, with each report written straight to disk; no subagent ever writes the plan file, invokes `/wrap-up`, runs the gate, or produces a verdict the orchestrator acts on; the orchestrator is the sole writer of the plan file, writes by matching the slice id (never a line number), re-reads before each write, and **aborts** if the heading text changed under it rather than attempting to reconcile a concurrent hand-edit. (4) **Failure**: any red, any blocked subagent, any missing check → halt immediately, leave the tree exactly as it is, never retry, report the slice id and the failing command. (5) **Resume**: on re-invocation, reconstruct completed slices from `git log` (not from the markers), refuse to resume on a dirty tree, and tell the user to commit their fix or `git reset --hard` first. (6) **Commits, never pushes.** End with the single breadcrumb line pointing at `/wrap-up`. Verify: read back `skills/run-plan/SKILL.md` and confirm every one of: up-front plan validation with named-slice refusal; the two-subagent Check/Build split; orchestrator-only gate invocation; the three postflight assertions including the untouched-check diff; sole-writer + id-matched + abort-on-external-edit; halt-no-retry; resume-from-git-log + refuse-dirty; commit-not-push; ` ✅` provenance stamp; and the breadcrumb.
