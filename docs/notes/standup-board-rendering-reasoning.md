# Standup Board Rendering — reasoning

**Decision:** Do both, split by cost. Make `/standup`'s *default* output a fuller multi-plan
table (ephemeral, all active plans — not just WIP + the one NEXT), and add an *explicit*
`/standup --board` that regenerates a persisted board file on demand. The persisted file is
written only on the flag (never on a routine standup), carries a "generated — do not edit"
header, and must be excluded from `/standup`'s own plan sweep so it can't self-parse.

**Why:** The two halves of the idea have very different cost profiles. A richer ephemeral render
is nearly free and drift-proof, so it should just be the default. A persisted artifact is what
the user actually keeps reaching for (they hand-maintain `docs/plans/00-master-index.md` in
workbench), but writing it on *every* standup churns git and re-introduces the drift the system
avoids — so it belongs behind an explicit flag, regenerated from the plan files each time.

## Options considered

- **A — Ephemeral-only (richer default render).** Extend `/standup` to print a full table of all
  active plans. *Fails:* not shareable, gone when the terminal scrolls, doesn't match the
  keep-a-board habit that prompted this. Solves the glance problem, not the artifact problem.
- **B — Persisted `docs/plans/BOARD.md`, regenerated every standup.** *Fails:* (1) lands *inside*
  the dir `/standup` scans (`docs/plans/*.md`, only `_done/` skipped) → self-parses as a plan
  unless explicitly excluded; (2) committed + rewritten every run → git churn on every standup;
  (3) a persistent editable file tempts hand-edits → drift returns.
- **C — Both, split by cost (chosen).** Richer table becomes the default (A's win, no file);
  persisted board is opt-in via `--board` (B's win, no per-run churn). *Why it wins:* gives the
  cheap glance always and the shareable artifact only when asked, and the explicit-flag gate is
  the natural place to hang the "generated, regenerated, excluded-from-sweep" guarantees that
  neutralize B's three failure modes.

## Risks / open questions carried into the plan

- **Parse-collision (must handle):** the persisted board file must be excluded from step-1's
  `docs/plans/*.md` sweep, the same way `_done/` is skipped — or it self-parses as a plan.
  Location choice is part of this: `docs/plans/BOARD.md` (skip by name) vs `docs/BOARD.md`
  (outside the sweep entirely). Lean toward outside-the-sweep to make the exclusion structural,
  not a rule that can be forgotten.
- **Git churn:** write the file only on `--board`, never on a routine standup. Decide whether it's
  committed or gitignored (a shareable artifact probably wants to be committed, but then only when
  the user chooses to snapshot).
- **Drift-via-hand-edit:** the file needs a "generated — regenerated by `/standup --board`, do not
  edit" header; hand edits are overwritten on next run. The plan should state this contract.
- **Columns / "status details" per row:** define the row schema — plan, status, slices done/total,
  next open slice, last-touched, model/effort. Defer exact set to the plan.
- **Relationship to workbench's hand-built `00-master-index.md`:** decide whether `--board` is
  meant to converge that per-project pattern or coexist with it. Not blocking, but name it.
- **Skill vs flag:** `--board` on `/standup` vs a separate tiny skill. Lean flag (shares the same
  plan-parsing pass standup already does; a separate skill would duplicate it).
